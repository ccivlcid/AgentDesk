# 직원 살펴보기(Heartbeat) 로그·가이드 개선 계획

## 1. 직원 선택이 오피스팩 직원으로 안 나오는 문제

**원인**: 직원관리(AgentManager)는 `managerAgents`(현재 팩 전용)를 쓰는데, Heartbeat 패널은 `displayAgents`(merged = 팩 + 일부 글로벌)를 받고 있어서 다른 팩/글로벌 직원이 섞여 보일 수 있음.

**조치**: Heartbeat 패널에 `managerAgents`를 넘기도록 변경. → **적용 완료**

---

## 2. 가이드: 모달 → 접기/펼치기

**요구**: 가이드 모달 대신 화면 안에서 접었다 펼 수 있는 형태.

**조치**:
- (?) 버튼 클릭 시 모달 열기 제거.
- 본문 상단에 "직원 살펴보기 가이드" 접기/펼치기 블록 추가.
- 펼치면 기존 가이드 내용(단계 설명 + 팁) 표시.

---

## 3. 최근 로그 부담 완화 및 로그 내용 표시

### 3.1 로그 삭제

- **API**: `DELETE /api/heartbeat/logs/:id` (단건 삭제). 추후 필요 시 `?before=ts` 또는 `?ids=1,2,3` 등 일괄 삭제 확장 가능.
- **UI**: 각 로그 행에 삭제 버튼(휴지통). 클릭 시 확인 후 삭제하고 목록 갱신.

### 3.2 로그 내용 표시

- **현재**: 상태 아이콘, 에이전트, 한 줄 요약(정상/Normal 또는 첫 번째 finding 메시지), 시간만 표시.
- **개선**: 각 행을 클릭하거나 "자세히" 토글 시 펼쳐서 표시:
  - `summary` (전체 요약)
  - `findings` 배열 전체 (check, severity, message, details)
  - 필요 시 `created_at` 포맷 표시
- **UI**: 로그 행에 접기/펼치기(chevron) 추가, 펼친 영역에 로그 상세 렌더.

### 3.3 구현 순서

1. 서버: `DELETE /api/heartbeat/logs/:id` 추가.
2. 클라이언트: `deleteHeartbeatLog(id)` API 함수 추가.
3. HeartbeatPanel: 로그 행에 펼치기/접기 + 상세 블록, 삭제 버튼 + 확인 후 호출 및 refresh.
