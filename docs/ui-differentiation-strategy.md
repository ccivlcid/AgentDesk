# UI/UX 차별화 전략 및 사용자 경험 개선 가이드

> **배경**: AgentDesk는 [claw-empire](https://github.com/GreenSheep01201/claw-empire) 오픈소스를 fork해서 시작했습니다.
> 픽셀아트 오피스 뷰, 기본 사이드바 구조, 에이전트/태스크 기반 아키텍처 모두 원본에서 가져왔습니다.
> 이 문서는 AgentDesk만의 정체성을 확립하기 위한 전략을 정리합니다.

---

## 1. 현황 파악 — 무엇이 claw-empire이고, 무엇이 우리 것인가

### claw-empire에서 가져온 것 (원본)
- PixiJS 기반 픽셀아트 오피스 뷰 (에이전트 이동, 부서 구조)
- 사이드바 네비게이션 구조
- 기본 에이전트/부서 관리 UI 패턴
- 태스크 보드 (칸반) 기반 구조
- 전체 앱 레이아웃 셸 (헤더 + 사이드바 + 콘텐츠)
- "Claw Empire" 컨셉 및 CEO/직원 스토리텔링

### AgentDesk가 추가로 구축한 것 (우리 것)
포크 이후 **39,000줄 이상, 158개 파일** 변경/추가:

| 카테고리 | 추가된 기능 |
|---|---|
| **오피스 커스터마이즈** | 룸 테마, 시즌 파티클, CEO 커스터마이즈, 가구 배치 |
| **그리기 스타일** | business / cyber / retro / pixel / default 5가지 DrawerStyle |
| **라이브러리 시스템** | Agent Rules, Hooks, Memory 라이브러리 (학습/이력 관리) |
| **스킬 시스템** | Skills Library, 학습 모달, Custom Skill, 내보내기/가져오기 |
| **대시보드** | Hero KPI 섹션, Ops 섹션, CollapsibleSection |
| **결과물 관리** | Deliverables, Git 연동, 협업자 섹션 |
| **프로젝트 관리** | Burndown Chart, Project Editor, Insights Panel |
| **태스크 고도화** | Gantt Chart, 스케줄러, 태스크 의존성 블로킹 |
| **게임룸** | 테트리스, 틱택토, 메모리 매치, 가위바위보 |
| **설정** | API/CLI/Gateway/OAuth/Data 탭별 설정 패널 |
| **알림/채팅** | NotificationCenter, Decision Inbox, StreamingMessage |
| **워크플로우 팩** | 개발/마케팅 등 팩 선택, Seed Agent 자동 생성 |
| **다국어** | ko/en/ja/zh 전체 번역 |

## 2. 문제의 본질

기능은 크게 확장했지만, **시각적 레이어는 여전히 claw-empire와 동일**합니다.

사용자가 처음 화면을 봤을 때 AgentDesk와 claw-empire를 구분할 수 없습니다.
차별화는 기능 추가만으로는 안 되고, **보이는 것**이 달라야 합니다.

핵심 과제:
1. claw-empire의 비주얼을 그대로 쓰면서도 AgentDesk 고유 아이덴티티를 만들기
2. 우리가 추가한 기능들(라이브러리, 스킬, 결과물 등)을 UI에서 앞으로 꺼내기
3. 사용자가 "이건 claw-empire 아닌가?" 가 아니라 "이건 AgentDesk구나"를 느끼게 하기

---

## 3. 차별화 전략

### 3-1. 브랜드 레이어 교체 (즉시 가능)

claw-empire의 비주얼 코드를 쓰더라도, 브랜딩을 완전히 교체하면 인식이 달라집니다.

| 요소 | claw-empire | AgentDesk 방향 |
|---|---|---|
| 로고 | Claw 아이콘 | AgentDesk 고유 픽셀아트 로고 |
| 앱 이름 표시 | "Claw Empire" | "AgentDesk" + 회사명 |
| 색상 팔레트 | 기본 slate | `empire-*` 토큰 전면 적용 + 포인트 컬러 |
| 폰트 | 시스템 폰트 | Sora (제목) + IBM Plex Sans KR (본문) 100% 적용 |
| 빈 화면 | 기본 텍스트 | AgentDesk 전용 픽셀아트 일러스트 + 상황별 대사 |
| 로딩 화면 | 기본 스피너 | 픽셀아트 에이전트 걷는 애니메이션 |

### 3-2. 우리가 만든 기능을 전면에

현재 우리가 만든 가장 독창적인 기능들이 UI 깊숙히 묻혀있습니다.

**지금 묻혀있는 것들:**
- Agent Rules / Hooks / Memory 라이브러리 → 사이드바 "Library" 하위 메뉴 3번 클릭
- Skills + 학습 시스템 → Library 안에 또 숨겨짐
- Drawing Style 5종 → 오피스 뷰 설정 안에
- Gantt Chart → 태스크 탭 안에

**앞으로 꺼내는 방법:**
- 대시보드 홈에 "오늘의 에이전트 활동 요약 + 최근 스킬 학습 현황" 위젯
- 사이드바 Library 그룹에 배지(badge) — 새로 학습된 룰/스킬 개수 표시
- 오피스 뷰 에이전트 우클릭 → "스킬 현황 보기", "룰 확인" 빠른 접근
- 온보딩 시 스킬/룰 시스템 하이라이트

### 3-3. 오피스 뷰를 우리 것으로 만들기

픽셀아트 렌더링 자체는 claw-empire이지만, 우리가 그 위에 쌓은 것이 있습니다.
이걸 더 강화하면 "이건 AgentDesk"가 됩니다.

**우리가 이미 추가한 것:**
- Drawing Style 5종 (business / cyber / retro / pixel / default)
- 룸 테마, 가구 배치, CEO 커스터마이즈
- 시즌 파티클 (눈, 벚꽃 등)

**더 추가하면 차별화되는 것:**
- 에이전트 말풍선 — 현재 태스크 진행률, 현재 작업 내용 실시간 표시
- 부서 별 이벤트 이펙트 — 태스크 완료 시 해당 부서에 팡파레
- CEO 오피스 → AgentDesk 고유 공간 (claw-empire에 없는 레이아웃)

## 4. 사용자 측면 — 실제로 있으면 좋을 기능

### 4-1. 첫 경험 (Onboarding)

**현재:** 앱 열면 바로 빈 오피스. 뭘 해야 할지 모름.

**필요한 것 — "회사 설립 위저드":**
```
Step 1: 회사 이름 & CEO 이름 입력
Step 2: 워크플로우 팩 선택 (개발팀 / 마케팅팀 / 에이전시...)
Step 3: AI 제공자 연결 안내 (Claude / Gemini / Codex)
Step 4: 첫 에이전트 자동 생성 → 오피스 입장
```
이것만으로 첫 인상이 완전히 달라집니다.

### 4-2. 글로벌 검색 & 커맨드 팔레트

CLI 유저가 주 타겟이므로, 마우스 없이 이동할 수 있어야 합니다.

```
Cmd+K / Ctrl+K  → 커맨드 팔레트 열기
  - 에이전트 이름 검색 → 즉시 채팅
  - 태스크 검색
  - 빠른 액션: "새 태스크", "오피스 뷰로", "설정"

키보드 단축키:
  G + O  → 오피스
  G + D  → 대시보드
  G + T  → 태스크 보드
  N      → 새 태스크 생성
  Esc    → 현재 모달 닫기
```

### 4-3. 실시간 알림 강화

| 이벤트 | 현재 | 개선 |
|---|---|---|
| 태스크 완료 | 이벤트 없음 | 토스트 + 오피스 이펙트 |
| 에이전트 오류 | 없음 | 오피스에서 에이전트 빨간 표시 + 알림 |
| CLI 토큰 임계값 | 없음 | 사이드바 경고 배지 |
| 오래된 태스크 (Stale) | 없음 | 칸반 카드 흐림 처리 |

### 4-4. 에이전트 프로필 고도화

현재 에이전트 상세는 기본적인 정보만 보여줍니다.
사용자가 실제로 알고 싶은 것:

- **이 에이전트가 잘하는 게 뭔가?** → 완료 태스크 통계, 전문 스킬 목록
- **지금 뭘 하고 있나?** → 현재 태스크 + 서브태스크 실시간 진행률
- **오늘 얼마나 일했나?** → 일별 활동 히트맵 (GitHub contribution 스타일)
- **비용은 얼마나 썼나?** → CLI 토큰 사용량 per 에이전트

### 4-5. 태스크 생성 단순화

현재: 태스크 보드 뷰로 이동 → 생성 버튼 → 모달 열기.

**원하는 것:**
- 오피스 뷰에서 에이전트 클릭 → "업무 지시하기" → 인라인 입력
- 채팅창에서 `/task 내용` 명령어로 즉시 생성
- 음성 입력 (브라우저 Speech API) - 선택적

### 4-6. 생산성 리포트

하루/주 단위로 자동 생성:
```
📊 오늘의 AgentDesk 리포트
- 완료된 태스크: 12개
- 가장 활발한 에이전트: Claude-Dev (8 태스크)
- 토큰 사용량: 42,000 tokens (전일 대비 -15%)
- 병목 부서: QA팀 (5개 태스크 대기 중)
```

### 4-7. 모바일 반응형

현재 모바일 경험이 좋지 않습니다 (오피스 뷰 특히).

최소한 다음은 모바일에서 동작해야 합니다:
- 대시보드 KPI 확인
- 태스크 상태 변경
- 에이전트 채팅
- 알림 확인

## 5. 실행 우선순위

### Phase 1 — 브랜드 분리 (1-2주, 코드 변경 최소)
- [ ] AgentDesk 고유 로고 제작 (픽셀아트 스타일)
- [ ] 사이드바 브랜드 영역 교체 (로고 + 회사명)
- [ ] Sora + IBM Plex Sans KR 폰트 전 영역 일관 적용
- [ ] 빈 상태 화면에 전용 일러스트 + 대사 추가
- [ ] 로딩 화면 교체

### Phase 2 — UX 핵심 (2-4주)
- [ ] 온보딩 위저드 (회사 설립 플로우)
- [ ] `Cmd+K` 커맨드 팔레트
- [ ] 태스크 완료/오류 토스트 알림 강화
- [ ] 에이전트 말풍선 (오피스에서 현재 작업 표시)

### Phase 3 — 차별화 완성 (1-2달)
- [ ] 에이전트 프로필 고도화 (활동 히트맵, 스킬 현황)
- [ ] 오피스 뷰 → 태스크 직접 생성
- [ ] 일일/주간 생산성 리포트
- [ ] 모바일 핵심 기능 반응형

## 6. 피해야 할 것

- **claw-empire README 이미지/스크린샷** → 우리 앱으로 전량 교체 필요
- **generic Tailwind 클래스** (`blue-500`, `gray-700`) → 반드시 `empire-*` / `th-*` CSS 변수 사용
- **"기능을 추가하면 차별화된다"는 착각** → 보이는 것이 달라야 사용자가 인식함
- **깊이 묻힌 핵심 기능** → 우리가 만든 스킬/룰/메모리 시스템을 홈/대시보드에서 보여줄 것
